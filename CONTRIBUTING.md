# Contributing To MPP-I

Thank you for helping improve MPP-I.

## Contribution Types

Use Issues for actionable changes:

- errata,
- unclear wording,
- missing or stale citations,
- broken links,
- technical inconsistencies,
- implementation questions that can be turned into concrete work.

Use Discussions for open-ended topics:

- protocol design alternatives,
- inference engineering considerations,
- settlement or facilitator models,
- wallet behaviour,
- ecosystem adoption questions,
- future profile proposals.

Use Pull Requests for concrete changes to repository files.

## Pull Request Rules

Before opening a pull request:

1. Keep the change scoped.
2. Explain which issue or discussion it addresses.
3. Do not change the meaning of MPP-I without an issue or discussion first.
4. Do not add generated files, private review material, local scratch files, or unpublished reviewer comments.
5. For paper changes, update both the source and any derived readable docs only when the change is accepted.

## Protocol Change Process

Protocol changes should start as a Discussion unless they are minor editorial fixes.

A protocol change should explain:

- the problem,
- the proposed change,
- compatibility impact,
- security or privacy impact,
- settlement or accounting impact,
- whether it changes the paper, draft spec, schemas, examples, or libraries.

## Errata

Errata should include:

- location,
- current text,
- proposed correction,
- reason,
- source or citation if factual.

## Citation Changes

Citation changes should include:

- source title,
- URL or DOI,
- author or organisation,
- publication or access date,
- what claim the citation supports.

## Conduct

Participation is governed by `CODE_OF_CONDUCT.md`.
