# Executive Summary

MPP-I is a proposed MPP-compatible, inference-native payment profile for metered inference.

MPP and related Payment HTTP work make it possible for agents and applications to pay for API requests, tool calls, and content through an HTTP payment flow. Inference is a central resource for that flow. Agents need inference to reason, plan, generate, verify, and act, and they need a way to authorise bounded spend when they call external inference providers.

Inference does not behave like a fixed-price resource request. A provider can publish prices, bounds, and admission requirements, but the final delivered output length is not known when the request is admitted. The provider must admit prefill, allocate scheduler and KV-cache state, and run decode while output tokens are produced.

MPP-I defines the missing inference lifecycle inside an MPP payment envelope. It adds quotes, policy, run-scoped authorisation, cumulative vouchers or credit grants, signed meter frames, low-credit states, drain-entry behaviour, stop boundary rules, settlement references, and final inference receipts.
